# Pricing Engine Rules

- Time Window lifecycle and One-time Publish lifecycle must remain isolated.
- Never mix active-window with published semantics.
- Preserve storefront ownership semantics.
- Do not modify revert pipeline without explicit instruction.
- Prefer minimal targeted fixes over broad refactors.
- Preserve existing Polaris modal/layout patterns.
- Avoid unnecessary schema changes.
- Maintain operational readability for merchants.
